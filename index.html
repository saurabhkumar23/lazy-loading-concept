What Is Lazy Loading?

Lazy loading is the practice of delaying load or initialization of resources until they're actually needed to improve performance and save system resources. Lazy loading images means loading images on websites only when they appear in the browser’s viewport. This means that if users don’t scroll all the way down, images placed at the bottom of the page won’t even be loaded. All the media-heavy sites use this approach like unsplash.com, pexels.com and the social media websites too.

Techniques for implementing lazy loading:

#1 Native Lazy Loading

  • Only applies to images and iframes
  • just use the loading attribute with values (lazy or eager) to image and iframe tag. The loading attribute gives us the option to delay off-screen images and iframes until users scroll to their location on the page.
    • lazy: works great for lazy loading
    • eager: instructs the browser to load the specified content right away

#2 Lazy Loading Using the Intersection Observer API

  • The Intersection Observer API provides a way to asynchronously observe changes in the intersection of a target element with an ancestor element or with a top-level document’s viewport.
  • for images, the path is contained inside a data-src attribute, not a src attribute. The reason is that using src means the image would load right away, which is not what you want
